¿Qué es una Excepción?

Según P. J. Deitel, una excepción es una indicación de un problema que ocurre durante la ejecución de un programa.

  • El nombre "excepción" implica que el problema ocurre con poca frecuencia.
  • Permite crear programas robustos y tolerantes a fallas.
  • Separa el código de detección de errores de la lógica del negocio.
"La robustez es la capacidad de un sistema para manejar condiciones de error durante la ejecución."

Jerarquía de Clases Throwable

Throwable
Superclase de todas las excepciones y errores en Java.
Error
Problemas serios del sistema (JVM). No deben ser capturados.
Exception
Problemas que la aplicación puede capturar y manejar.

Nota: Las excepciones se dividen en Checked y Unchecked.

Estructura de Manejo

try { // Código propenso a errores } catch (Exception e) { // Manejo del problema } finally { // Limpieza obligatoria }

Try: Contiene el código que podría lanzar una excepción.

Catch: Captura y maneja la excepción.

Finally: Se ejecuta siempre (ideal para liberar recursos).

Ejemplo 1: ArithmeticException

Este código captura el error cuando intentas realizar una operación matemática imposible.
Jerarquía: Throwable → Exception → RuntimeException → ArithmeticException.

public class Ejemplo1 { public static void main(String[] args) { try { System.out.println("Intentando dividir 10 entre 0..."); int resultado = 10 / 0; } catch (ArithmeticException e) { System.err.println("CAPTURA: No se puede dividir entre cero."); System.err.println("Mensaje de Java: " + e.getMessage()); } } }
Intentando dividir 10 entre 0... CAPTURA: No se puede dividir entre cero.

Ejemplo 2: InputMismatchException

Ideal para demostrar cómo manejar cuando un usuario ingresa texto en lugar de un número.
Jerarquía: Throwable → Exception → RuntimeException → NoSuchElementException → InputMismatchException.

// Si el usuario ingresa una letra import java.util.Scanner; import java.util.InputMismatchException; public class Ejemplo2 { public static void main(String[] args) { Scanner teclado = new Scanner(System.in); try { System.out.print("Ingresa tu edad (número entero): "); int edad = teclado.nextInt(); System.out.println("Edad registrada: " + edad); } catch (InputMismatchException e) { System.err.println("ERROR: El dato ingresado no es un número válido."); } } }
Ingresa tu edad (número entero): Hola ERROR: El dato ingresado no es un número válido.

Ejemplo 3: Uso de Finally

Demuestra que ciertas instrucciones se ejecutan siempre, garantizando la robustez del programa.

try { System.out.println("Abriendo base de datos..."); throw new Exception("Fallo de conexion"); } catch (Exception e) { System.out.println("Atrapado: " + e.getMessage()); } finally { System.out.println("Cerrando base de datos forzosamente."); }
Abriendo base de datos...
Atrapado: Fallo de conexion
Cerrando base de datos forzosamente.

Ejemplo 4: Cláusula Throws

Demuestra cómo delegar la responsabilidad de un error de archivos (IOException) a quien llama al método.
Jerarquía: Throwable → Exception → IOException.

import java.io.FileReader; import java.io.IOException; public class Ejemplo4 { // El método avisa que puede fallar con 'throws' public static void abrirArchivo() throws IOException { FileReader fr = new FileReader("archivo_inexistente.txt"); } public static void main(String[] args) { try { abrirArchivo(); } catch (IOException e) { System.err.println("CAPTURA: El archivo solicitado no se pudo encontrar."); } } }
CAPTURA: El archivo solicitado no se pudo encontrar.

Ejemplo 5: Excepción Personalizada

Demuestra cómo extender la clase Exception para crear tus propios errores.

// Creamos nuestra propia clase de error class ErrorSaldoException extends Exception { public ErrorSaldoException(String msg) { super(msg); } } public class Ejemplo5 { public static void main(String[] args) { int saldo = 50; int retiro = 100; try { if (retiro > saldo) { throw new ErrorSaldoException("No tienes suficiente dinero."); } } catch (ErrorSaldoException e) { System.err.println("REGLA DE NEGOCIO: " + e.getMessage()); } } }
REGLA DE NEGOCIO: No tienes suficiente dinero.

Conclusiones

Separación
Separa el código de manejo de errores de la lógica principal.
Propagación
Las excepciones pueden propagarse hacia arriba en la pila de llamadas (Stack Unwinding).
Confianza
Un buen manejo incrementa la confianza del usuario en el software.
